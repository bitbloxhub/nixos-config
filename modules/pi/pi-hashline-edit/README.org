* pi-hashline-edit (declarative Nix packaging)

This directory stores reproducible npm metadata for packaging
~pi-hashline-edit~ as a local Pi extension via ~buildNpmPackage~.

** Files

- ~package.json~
  - runtime deps only (what extension imports at runtime)
- ~package-lock.json~
  - lockfile for deterministic npm dependency closure

** Wired from

- ~modules/pi/default.nix~
  - fetches source from GitHub tag
  - copies this directory's ~package.json~ + ~package-lock.json~ in ~postPatch~
  - pins ~npmDepsHash~

** Bump flow

1. Update version + source hash in ~modules/pi/default.nix~.
2. If runtime deps changed, update ~package.json~ here.
3. Regenerate lockfile from this directory:

#+begin_src bash
cd modules/pi/pi-hashline-edit
npm install --package-lock-only --ignore-scripts
#+end_src

4. Recompute ~npmDepsHash~ in Nix:
   - set temporary fake hash
   - run build
   - copy reported "got" hash into ~npmDepsHash~

This keeps install fully declarative and reproducible.
